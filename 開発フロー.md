# Antigravity Superpowers 開発ワークフロー標準

## 🌟 黄金の基本サイクル（新規開発・機能追加）
常にこの順番を守ることで、AIの暴走を防ぎ、手戻りを最小限にする。

1. `/brainstorm [作りたいもの]`
   - 目的：仕様の具体化と要件定義
   - 行動：AIからの質問に答え、曖昧な部分（エラー処理、UI、データ構造など）を潰す。
2. `/plan`
   - 目的：実装手順の設計（地図の作成）
   - 行動：AIに `plan.md` を作成させ、タスクが適切に細分化されているか確認・承認する。
3. `/build` （または `/tdd`）
   - 目的：計画に基づく一気呵成な実装
   - 行動：AIが「1ステップ実装 → テスト」を繰り返すのを監視する。より堅牢にしたい場合は、テスト駆動型の `/tdd` を使用する。

## 🛠️ シチュエーション別アプローチ

### 既存コードの改善・リファクタリング
1. `/examine` : 対象コードの構造と依存関係をAIに深く理解させる。
2. `/review` : コードの品質（可読性、保守性、脆弱性）を診断させる。
3. `/refactor` : 機能を変えずに、診断結果をもとにコードを最適化させる。

### バグ調査と修正
1. `/search` : エラーメッセージや関連キーワードから、原因になり得る箇所を特定。
2. `/debug` : 仮説立案〜検証を繰り返し、根本原因を修正。
3. `/test` : 修正によって他の機能が壊れていないか（リグレッション）を確認。

## 📝 開発をスムーズにするお作法
- **迷子防止:** 定期的に `/status` を実行し、現在地と残タスクを確認する。
- **記録保持:** 実装完了後、`/docs` でREADMEや仕様書を最新化する。
- **最終確認:** デプロイ・公開前に `/audit` を実行し、セキュリティリスクを排除する。
