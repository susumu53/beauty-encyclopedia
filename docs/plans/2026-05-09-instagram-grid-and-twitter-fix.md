# 設計書：Instagram風グリッド表示の実装とTwitter埋め込みの改善

## 1. 目的
- ブログ記事内の画像表示を、Instagramの投稿一覧のような「3列グリッド表示」に変更し、視覚的なリッチさを向上させる。
- Twitterの埋め込みが一部表示されていない問題を、ID判定ロジックの改善により解決する。
- 全てのスクレイパーで複数画像の取得をサポートし、表示枚数を増やす。

## 2. 変更内容

### 2.1 スクレイパーの改修 (`scraper.py`)
- `bi-girl.net` の一覧ページから、1枚だけでなくカード内に存在する全ての画像URLを取得するように変更する。
- 戻り値のキーを `image_url` (単数) から `images` (複数) に統一する。

### 2.2 メイン処理の改修 (`main.py`)
- **SNS判定ロジック**: 
    - ソース名によるブラックリスト方式を廃止。
    - `id` が英数字のハンドル名（Twitter IDとして有効な形式）であれば、Twitter埋め込みを実行する。
- **HTML/CSSの刷新**:
    - 画像表示セクションに `display: grid; grid-template-columns: repeat(3, 1fr);` を使用したスタイルを導入。
    - 1枚目の画像を大きく見せる「フィーチャーレイアウト」の採用。
    - 各画像に `aspect-ratio: 1/1; object-fit: cover;` を適用し、正方形に統一。
- **SNSリンクの装飾**:
    - テキストリンクから、アイコン付きのボタン風デザインに変更。

### 2.3 データの整合性確保
- 既存のキューファイル (`queue_*.json`) との互換性を保つため、`image_url` と `images` の両方を許容する読み込み処理を実装する。

## 3. 期待される効果
- Instagramのアカウントを持たないユーザーでも、ブログ上でInstagramのような最新投稿一覧を確認できるようになる。
- 画像枚数が増えることで、記事の滞在時間と満足度が向上する。
- Twitterの埋め込みが復活し、よりリアルタイムな情報提供が可能になる。
