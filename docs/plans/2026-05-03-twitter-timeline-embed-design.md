# 2026-05-03 X(旧Twitter)タイムライン埋め込み機能の導入

## 概要
現在、引用元のブログ（ReinaSex等）から取得している画像が、直リンク制限により表示されない問題が発生しています。
この問題を解決し、かつコンテンツをリッチにするため、X（旧Twitter）のタイムラインを記事内に直接埋め込む機能を導入します。

## 目的
- 画像が表示されない（リンク切れ）問題を、公式の埋め込み機能で解消する。
- 読者がブログ上で最新の投稿を確認できるようにし、回遊性や満足度を高める。
- SNSリンク、タイムライン、DMMアフィリエイトを組み合わせた「最強の記事テンプレート」を構築する。

## 変更内容

### 1. 記事テンプレートの刷新 (main.py)
以下の順序でコンテンツを構成するように変更します。
1. **アカウント情報**: 名前とID
2. **SNSリンク**: X, Instagram, TikTok へのリンク
3. **Xタイムライン埋め込み**: 公式ウィジェットを使用した最新投稿リスト
4. **DMM作品セクション**: DMM APIで取得した関連作品

### 2. 埋め込み仕様
- **タイプ**: プロフィールタイムライン (URL: `https://twitter.com/[screen_name]`)
- **高さ**: 600px (固定。記事が長くなりすぎないように調整)
- **テーマ**: ライト（デフォルト。必要に応じて将来的に切り替え可能）
- **スクリプト**: `https://platform.twitter.com/widgets.js` を記事末尾に挿入

### 3. 画像表示の廃止
直リンク制限により表示されないため、従来の `image_html` 部分は削除します。

## データフロー
1. スクレイパーが `id` (Xのスクリーンネーム) を取得。
2. `main.py` が `id` を用いて埋め込み用HTMLコードを生成。
3. Livedoor API を通じて記事を投稿。

## テスト計画
- `python main.py --dry-run` を実行し、出力されるHTMLコードに埋め込みタグが正しく含まれているか確認する。
- 実際に1件テスト投稿を行い、Livedoorブログ上でタイムラインが表示されるか、スクリプトが正しく動作するか確認する。

## 考慮事項
- 非常に稀にXアカウントが削除されている場合、埋め込み枠には「Tweets by...」というリンクのみが表示されます。
- `widgets.js` は1つの記事に複数回含まれてもブラウザ側で最適化されますが、念のため各埋め込みの直後に配置します。
