# DMMアフィリエイト連携機能 設計ドキュメント

## 1. 目的
ブログ記事に、各モデルのDMM作品（一般・成人向け両方）を画像リンクとして自動挿入し、収益化を強化する。

## 2. 外部API
- **DMM アフィリエイト API (v3)**
  - `https://api.dmm.com/affiliate/v3/ItemList`
  - 使用パラメータ:
    - `api_id`: ユーザー提供のAPI ID
    - `affiliate_id`: ユーザー提供のアフィリエイトID
    - `site`: `DMM.com` および `FANZA`
    - `keyword`: モデル名（名前）
    - `output`: `json`

## 3. システム構成
### 3.1 新規モジュール: `dmm_client.py`
- `DMMClient` クラスを定義。
- `search_items(keyword)` メソッド:
  - `DMM.com` と `FANZA` の両方にリクエストを送信。
  - 重複を除去し、最大4件までの作品情報を返す（画像URLとアフィリエイトURL）。

### 3.2 メイン処理の拡張 (`main.py`)
- モデルごとのループ内で `DMMClient.search_items(name)` を呼び出し。
- 取得したアイテムをHTMLの `<img>` タグ群に変換し、既存のコンテンツに追記。
- 環境変数 `DMM_API_ID`, `DMM_AFFILIATE_ID` から認証情報を読み込む。

## 4. 表示デザイン
- **形式**: 画像のみ（クリックでDMMへ遷移）。
- **配置**: 横並びまたはタイル状。
- **スタイル**:
  - `max-width: 45%;` 程度のサイズで、モバイルでも2列に並ぶように調整。
  - 画像間に適切な余白を設定。

## 5. エラーハンドリング
- APIキーが未設定の場合は、DMM検索をスキップして続行。
- APIエラーやタイムアウトが発生した場合も、ログを出力して記事作成自体は継続（DMMリンクなしで投稿）。

## 6. テスト計画
- `tests/test_dmm_client.py` を作成。
- 各サイト（DMM/FANZA）のモックレスポンスを使用して、正しくマージ・抽出できるか検証。
- `main.py` の統合テストで、DMMセクションが正しくHTMLに挿入されるか確認。
